## Grounding

I used the local NMP architecture rules as the hard boundary: Rust owns policy/state, native shells render and execute raw capabilities, projections crossing FFI stay screen-shaped, provider and clock inputs must be replayable, and D8 forbids polling/sleep loops. The architecture scanner currently reports existing Pod0 D4/D6/D7/D8 findings; those are scenario targets and issue gates, not waivers.

Cassette verifier changes needed: add `recorded_at`, `expires_at` or `max_age_days`, failure class, provider model/version, request fingerprint, redaction inventory, scenario refs, doctrine refs, latency budget, and `no_live_fallback: true`. Following the service-virtualization guidance, cassettes older than 30 days should fail validation unless explicitly marked as a contract fixture with a newer contract-drift check.

## Catalog And Report Generator Changes

1. Catalog pages:
   - Add a sixth cluster file under `docs/testing/scenarios/catalog/` for the 102 new rows, or expand the existing five cluster files without changing their parser format.
   - Update `catalog/INDEX.md` totals from 320 to 422 and add a user-facing category rollup matching the 15 categories above.
   - Keep every row parseable by `catalog.py`: one `Given ..., when ..., then ...` sentence and semicolon-separated evidence keys.
   - Add optional `Issue:` and `Replay:` evidence keys. The current parser preserves unknown keys, so the generator can consume them after the record model is extended.

2. Evidence records:
   - Add required evidence kinds per scenario: screenshot, UI tree, performance metric, cassette, accessibility, Liquid Glass, NMP/chirp parity, issue link, and gh-pages page screenshot.
   - Require fresh per-screen screenshots, not cropped collages, for onboarding, settings, player, agent, voice, and Liquid Glass surfaces.
   - Require UI-tree dumps to include labels, roles/traits, values, selected/busy state, modal focus state, and stable accessibility identifiers.

3. Provider cassette reporting:
   - Extend `provider_cassettes_report.py` from "current fixtures table" to a matrix by provider, operation, success/failure class, freshness, mapped scenario refs, and no-live-fallback status.
   - Publish unmapped refs and missing required cassette families as blockers on `provider-cassettes/index.html`.
   - Include cassette freshness and contract-drift status in `data/rollups.json`.

4. Scenario report generation:
   - Add rollups for user-facing category, missing screenshot/UI-tree/metric/cassette evidence, issue gate status, Liquid Glass coverage, accessibility coverage, and architecture-scan doctrine coverage.
   - Surface `Issue:` evidence in `readiness.blocking_gates`, `instrumentation_gaps`, and the issues rollup.
   - Preserve generated scaffold pages as `incomplete` until all required evidence kinds are present and defects have issue links.
   - Add a gh-pages Playwright smoke gate for desktop/mobile report pages: no broken images, all screenshots have alt text/dimensions, no layout shift/text overlap, provider matrix renders.

5. Skill grounding:
   - Replace the older hardcoded single query in `contract.py` with a generated `data/skill-grounding.json` containing the exact queries and selected skills listed in this plan.
   - Show grounding on every scenario page and the category/provider rollup pages.
