# Experience North Star

Podcastr should feel like a refined audio product first and an agentic knowledge product second. The user should not have to manage AI machinery to enjoy a podcast, but the intelligence should be immediately available when they need it.

## Principles

- Calm by default, alive on demand.
- Playback is always trustworthy and close at hand.
- Answers should cite playable moments, not just text.
- Voice is a first-class control surface.
- Generated knowledge should feel like an editorial layer over the user's library.
- The app must work well while walking, driving, cooking, or using AirPods.
- Liquid Glass and motion should clarify hierarchy, not decorate around weak workflows.

## UX Test

For any major screen, ask:

- Can the user play, pause, search, and recover context quickly?
- Can the agent move the app to the relevant episode or timestamp?
- Can the user inspect where an answer came from?
- Does the screen still work with Dynamic Type, VoiceOver, and reduced motion?
