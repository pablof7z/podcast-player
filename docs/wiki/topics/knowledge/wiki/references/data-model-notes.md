# Data Model Notes

The current app template stores small app state in a JSON blob. That pattern is acceptable for settings, friends, notes, and bounded activity logs. It is not suitable for transcripts, wiki pages, embeddings, or episode libraries.

## Recommended Split

- `AppState` JSON remains for settings, ACLs, friends, agent memories, activity logs, and small UI state.
- SwiftData stores podcasts, episodes, transcript metadata, wiki pages, briefing scripts, downloads, and listening history.
- A separate sqlite-vec database stores transcript and wiki vectors plus full-text indexes.
- Vector data is derived and can be rebuilt from transcripts and wiki pages.
- A small Now Playing snapshot can remain in App Group `UserDefaults` for widgets.

## Implementation Notes

- Keep stable UUIDs across SwiftData and vector rows.
- Scope vector queries with podcast and episode ids rather than separate per-show indexes.
- Delete and reinsert wiki vectors when a wiki page recompiles.
- Treat embedding model changes as a background reindex with progress.
- Avoid putting transcript chunks into `AppStateStore` mutations.
